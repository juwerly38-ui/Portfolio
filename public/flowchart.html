5. 통역사 자동 배정 알고리즘 (approveRes)

bestOf 알고리즘: 해당 날짜에 이미 배정된 회의 시간(누적 분)이 가장 적은 통역사를 우선 선택합니다. 동일한 경우 배열 순서가 빠른 통역사가 선택됩니다. 이를 통해 통역사 간 업무 부하를 균등하게 분산시킵니다.

7. 정례 회의 등록 흐름

📌 정례 등록 흐름

  • 1

    신청자가 정례 탭에서 "협의 필요" 슬롯 클릭

  • 2

    정례 모달에서 회의명 · 기간 · 요일 · 시간 · 형태 · 방향 입력

  • 3

    3개월 이내 날짜 자동 생성 (공휴일 자동 제외)

  • 4

    관리자가 정례 그룹 단위로 승인 / 거절

  • 5

    승인 시 생성된 날짜 전체가 스케줄에 일괄 반영

🚫 수동 블록 기능

  • 1

    관리자가 특정 요일 + 시간을 수동 블록 지정

  • 2

    블록된 슬롯은 정례 시간표에서 노랑 잠금으로 표시

  • 3

    신청자는 해당 슬롯 클릭 불가 (이유 툴팁 표시)

  • 4

    관리자는 블록 해제 가능

8. 통역사 가능시간 등록 흐름

미등록 상태: 가능시간을 전혀 등록하지 않은 경우, 전체 슬롯이 available(기본 오픈) 상태로 표시됩니다. 하나라도 등록하는 순간부터 등록된 범위 밖은 자동으로 unavailable로 처리됩니다.

Reservation (신청)

id고유 ID
statuspending / approved / partial / rejected
direction통역 방향
isEmergency긴급 자동 전환 여부
autoEmergencyReason자동 전환 사유
meetingFormatremote / onsite
meetingTypegeneral / education / large
assignedInterpId배정 통역사 ID
현재 구현 방식: 모든 데이터는 STATE 전역 객체(JavaScript 메모리)에 저장됩니다. 페이지를 새로 고침하면 샘플 데이터로 초기화됩니다. 실서비스 전환 시 백엔드 API + DB 연동이 필요합니다 (각 함수 주석에 연동 포인트 명시).
