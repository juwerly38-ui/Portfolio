Background

왜 이 프로젝트를 시작했나

People Operations팀에서 프리랜서 통번역사 관리 및 임직원 통번역 서비스 운영을 전담하며, 반복되는 수작업과 데이터 단절 문제를 직접 목격했습니다. 개발팀의 리소스를 기다리지 않고 AI 도구를 활용해 직접 솔루션을 기획·구현하기로 결정했습니다.

Problem 01
📞 섭외 가능여부 확인 — 모두 수동
통번역사 섭외 가능 여부 확인이 시스템화되지 않아 매번 담당자를 통해 개별 연락·확인하는 방식으로 운영. 응답 대기 시간이 길고 실수 발생 위험이 높았음.
Problem 02
📊 통번역사 스케줄 — 엑셀 수기 관리
전체 통번역사의 스케줄을 전담 담당자가 엑셀로 수기 관리. 실시간 현황 파악 불가, 누락·중복 배정 위험, 히스토리 분석 불가능한 구조였음.
Problem 03
📝 결재 플랫폼 활용 — 업무 특화된 전용 폼 부재
통역 지원 신청에 결재 플랫폼을 활용 중이었으나, 통역 업무 특성에 맞는 필드(방향·형태·Zoom 링크 등)를 체계적으로 입력할 수 있는 전용 폼이 도입되지 않은 상태였음.
🤖 Core Approach
비개발 직군이 AI를 활용해
직접 시스템을 기획하고 구현
코딩 지식 없이도 ChatGPT, Claude 등 생성형 AI와의 대화를 통해 업무 요구사항을 HTML/JavaScript 프로토타입으로 직접 구현했습니다. AI를 단순 검색 도구가 아닌 기획·개발 파트너로 활용한 사례입니다.

  • · 날짜 선택 시 당일 예약 현황 즉시 표시

AI 협업 방식 — 구체적 사례

요구사항을 자연어로 정리 → AI가 기술 명세로 변환
"통역사마다 가능한 시간이 다르고, 회의 후 준비 시간이 필요해요" → AI가 슬롯 상태 판정 로직(7단계), 데이터 구조, 코드로 구현
비즈니스 규칙을 대화로 구현 — 긴급 신청 D+3 버퍼 로직
"긴급 신청은 D+3 영업일까지만 가능하고, 한국 공휴일은 제외해야 해요" → AI가 공휴일 DB, 영업일 계산 함수, 자동 잠금 UI를 코드로 구현
실무 피드백 → AI와 함께 즉시 반영
관리자·통역사 피드백 8건을 AI에게 전달하여 구현 여부 분석, 우선순위 분류, 개선 방향 제시까지 AI와 협업으로 처리
기획서도 AI와 함께 — Mermaid 순서도 포함 HTML 문서 제작
개발자 전달용 기획서를 AI와 협업해 14개 섹션, 7개 순서도, 전체 API 명세 포함 HTML 기획서로 직접 제작
📌 향후 전환 계획

현재 프론트엔드 프로토타입을 실서비스로 전환 시 Cloudflare Workers + D1 DB + 인트라넷 신청서 API 연동 구조를 기획 완료. 통역 신청서(Project 01) 제출 시 이 시스템에 자동 반영되는 통합 운영 플랫폼을 목표로 개발팀 협업을 준비 중.
